В Java "класс как модуль" поддерживается в первую очередь с помощью интерфейсов, позволяющих
определить контракт для объекта, ап также наследования. Можно написать контракт в качестве
интерфейса, либо использовать любой класс в качестве поставщика контракта. Интерфейсы и классы можно
расширять посредством наследования. В наследниках мы объявляем дополнительные методы. В
классах-наследниках также можно производить перегрузку методов с другими сигнатурами.
